TRANSPARENCY FOR MEDICAL SUPPLIES

BACKGROUND

Since COVID-19 has hit the US, medical supplies and devices have been in high demand. There has been much uncertainty about what medical supplies are readily available. This project aims to provide trust between the nation's suppliers and their buyers by creating a product specific network which allows everyone in the network to see the surplus or demand for a product. Using data analysis to create algorithms for demand of a product would also decrease the nation's dependency on the government to mandate production. By implementing a blockchain solution to this problem, along with RFID technology to scan each product into the surplus, the network can be certain on the availability of product.

CONTRACT

contract sellerContract{
//init seller
struct Seller{
string name;
uint quantity;
}
uint private supply;
//declare seller obj
Seller public seller_obj;
Seller[] public sellers;

//GET
function getSeller() public view returns (string memory, uint) {
return (seller_obj.name, seller_obj.quantity);
}

//SET
function setSeller(string memory name, uint quantity) public {
seller_obj = Seller({name:name, quantity: quantity});
supply = supply + quantity;
}

//GET SUPPLY
function getSupply() public returns (uint){
return supply;
}
//REMOVE FROM SUPPLY
function remove(string memory name, uint quantity) public {
seller_obj = Seller({name:name, quantity: quantity});
supply = supply - quantity;
}
}

HOW IT WORKS

From the "Home" page the supply of a specific medical item is shown.(getSupply()) A supplier can then navigate to the "Add" page to add to the supply.(setSeller()) This prototypes the scanning of RFID chips into the blockchain. To remove from the supply, a supplier can navigate to "Remove". This prototypes the selling of the items where an RFID scannig would remove the items from the supply.

STAKEHOLDERS

Hospitals and manufacturers would benefit from this implementation. Knowing how much of a product is available lets a hospital know whether they can obtain the product quickly or if they need to demand more be made. The solution also benefits manufacturers as they can know when to slow production. From a higher level, the nation's people and government will also benefit from this as they can trust products are being produced at a necessary rate to keep hospitals stocked as necessary.

THE LAST MILE

The last mile problem in this project is verifying the number of products in the surplus. While RFID scanning helps to eliminate entering duplicates to the surplus, there is nothing in place to evaluate whether the product is whole or if the manufacturer is just scanning RFID chips not associated to the product. One possible solution to this problem is validating a picture of the product along with the RFID scanning before it is added to the surplus. This solution still has problems as image processing cannot identify whether or a not a product works as intended but it does help verify that a product is in the surplus, broken or unbroken.
